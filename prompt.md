# AI에게 코딩 도움 받기 — 초보자 프롬프트 작성 지침

> "어떻게 말해야 AI가 정확히 만들어 줄까?"를 알려주는 가이드입니다.
> 기술 용어를 몰라도 됩니다. 상황별로 **복사해서 쓸 수 있는 템플릿**을 제공합니다.

---

## 먼저 알아두면 좋은 것 — "좋은 프롬프트"의 3가지 조건

1. **현재 상태를 말한다** — 지금 어떤 화면이 있고, 어떤 파일이 있는지
2. **원하는 결과를 말한다** — 만들고 나서 어떻게 동작해야 하는지
3. **제약 조건을 말한다** — 어떤 기술을 써야 하는지, 바꾸면 안 되는 것은 무엇인지

## [상황 1] 새로운 버튼을 추가하고 싶을 때

### 📋 템플릿

```
[파일명] 파일에 버튼을 추가해줘.
버튼 위치: [어디에 넣을지]
버튼 이름: "[버튼에 표시될 텍스트]"
클릭하면: [어떤 일이 일어나야 하는지]
스타일: [색상, 크기 등 원하는 모양]
```

### ✅ 좋은 예시

```
TeacherMode.jsx 파일 상단 바에 버튼을 추가해줘.
버튼 이름: "인쇄하기"
클릭하면: 현재 캔버스 화면이 인쇄 미리보기로 열려야 해.
스타일: 회색 outline 버튼으로, 기존 "개념 지도 저장" 버튼 왼쪽에 배치해줘.
ShadCN Button 컴포넌트 써줘.
```

### ❌ 나쁜 예시

```
버튼 만들어줘.
```
> 어떤 파일인지, 어디에 넣을지, 무엇을 하는 버튼인지 아무것도 없음

```
이 화면에서 나가는 방법: [뒤로 가기 버튼 등]
```

### ✅ 좋은 예시

```
현재 App.jsx는 home/teacher/student 세 가지 mode 상태로 화면 전환하는 구조야.
```

## [상황 3] 입력창(팝업)을 띄우고 싶을 때

### 📋 템플릿

```

팝업 제목: "[제목]"
입력 필드:
  - [필드 이름]: [어떤 값을 입력받는지], placeholder: "[안내 문구]"
  - (필드가 여러 개면 계속 추가)
확인 버튼을 누르면: [어떤 일이 일어나야 하는지]
취소 버튼을 누르면: 팝업이 닫힌다.
ShadCN Dialog 컴포넌트를 써줘.
```

### ✅ 좋은 예시

```
"학생 추가" 버튼을 클릭하면 입력창 팝업이 뜨게 해줘.
팝업 제목: "학생 등록"
입력 필드:
  - 이름: 학생 이름 입력, placeholder: "이름을 입력하세요"
  - 학번: 숫자만 입력, placeholder: "학번 6자리"
확인 버튼을 누르면: 입력한 학생 정보가 화면 아래 목록에 추가돼야 해.
취소 버튼을 누르면: 팝업이 닫히고 입력 내용은 사라져.
ShadCN Dialog 컴포넌트를 써줘.
```

```

저장 위치: [localStorage / Zustand 메모리 중 어느 것]
불러오는 시점: [언제 불러올지 — 앱 시작 시? 특정 버튼 클릭 시?]
저장할 데이터 구조 예시:
  { 필드명: 값, 필드명: 값 }
```

### ✅ 좋은 예시

```
학생이 만든 개념 지도를 저장하고 싶어.
저장 시점: "임시저장" 버튼을 클릭할 때
저장 위치: localStorage (새로고침해도 남아있어야 함)
불러오는 시점: 학생 모드 화면에 들어올 때 자동으로 불러와야 해.
저장할 데이터:
  {
    studentNodes: [...],  ← 학생이 배치한 노드들
    studentEdges: [...],  ← 학생이 연결한 관계들
    savedAt: "날짜"
  }
현재 Zustand store는 useMapStore.js에 있어.
```

## [상황 13] 현재 코드를 설명받고 싶을 때

### 📋 템플릿

```
아래 코드가 어떻게 동작하는지 설명해줘.
코드 위치: [파일명, 몇 번째 줄]
내가 이해 안 되는 부분: [특히 모르겠는 것]
[코드 붙여넣기]
```

### ✅ 좋은 예시

```
아래 코드가 어떻게 동작하는지 설명해줘.
코드 위치: useMapStore.js 전체
내가 이해 안 되는 부분:
  - persist가 뭘 하는 건지
```
