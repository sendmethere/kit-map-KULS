# Agent Teams — 발표 자료

> Claude Code의 멀티 에이전트 팀 기능 소개

---

## [1페이지] Agent Teams란 무엇인가

### 한 줄 정의

- Claude 한 명이 일하는 것이 아니라 **여러 Claude가 역할을 나눠서 동시에 협력**하는 기능
- 각 팀원은 독립적인 Claude 인스턴스 — 서로 다른 관점으로 같은 문제를 분석

### 왜 필요한가

- 복잡한 문제는 한 방향으로만 보면 놓치는 것이 생김
- 코드 작성자 / 검토자 / 사용자 관점이 동시에 필요한 경우
- 병렬 탐색이 필요한 작업에서 시간을 크게 단축

### 가장 효과적인 상황

- 코드 리뷰 (작성자 + 검토자 동시 투입)
- 버그 원인 조사 (여러 파일을 동시에 탐색)
- 설계 아이디어 비교 (UX 관점 vs 아키텍처 관점)
- 문서 작성 + 사실 검증 병렬 진행

## [2페이지] 팀 구성 예시

### 추천 팀 규모

- 최소 2명 / 권장 3~5명 / 너무 많으면 조율 비용 증가

### 교육 앱 개발 팀 예시

- **UX 담당** : 사용자 흐름, 화면 설계, 접근성 검토
- **아키텍처 담당** : 컴포넌트 구조, 상태 설계, 성능 고려
- **검토자** : 두 결과를 비교하고 최종 방향 결정

### 코드 품질 팀 예시

- **작성자** : 기능 구현 코드 작성
- **보안 검토자** : 입력값 검증, 취약점 탐색
- **테스트 작성자** : 엣지 케이스 발굴, 테스트 코드 생성

## [3페이지] 시작하는 방법 — 5단계

### 1단계: 기능 활성화

- `~/.claude/settings.json` 파일에 아래 항목 추가

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

- 또는 터미널 환경변수로 설정

```bash
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
```

### 2단계: 팀 생성 요청

- Claude에게 **자연어로 요청**하면 자동으로 팀원 생성 및 역할 배정

```
"3명의 팀원으로 팀을 만들어줘 — UX 담당, 아키텍처 담당, 검토자"
```

- Claude가 팀원을 만들고 작업 조율을 시작함

## [4페이지] 팀원 제어 방법

### In-process 모드 (기본)

- 별도 프로그램 설치 없이 현재 Claude Code 창 안에서 사용
- `Shift + ↓` 키로 팀원 순환 이동
- 선택된 팀원에게 직접 메시지 입력 가능

### 분할 창 모드 (고급)

- **tmux** 또는 **iTerm2** 필요 (터미널 분할 도구)
- 각 팀원이 별도 창에서 독립적으로 실행
- 여러 창을 동시에 보면서 각 팀원의 진행 상황 모니터링 가능

### 작업 흐름

- **리더(나)가 작업 할당** → 각 팀원에게 역할별 지시
- **팀원이 자체 요청** → 팀원끼리 필요한 정보를 주고받음
- **공유 작업 목록**으로 중복 작업 자동 방지

---

## [5페이지] 마무리 및 핵심 요약

### 정리 명령

- 작업 완료 후 반드시 정리

```
"Clean up the team"
```

- 생성된 팀원 프로세스와 리소스가 모두 해제됨

### 핵심 요약

- Agent Teams = **병렬 사고**를 Claude에게 시키는 기능
- 한 문제를 여러 관점에서 동시에 탐색 → 누락 최소화
- 가장 효과적인 작업: 코드 검토 / 버그 조사 / 설계 비교 / 문서 작성
- 권장 팀 규모: **3~5명**
- 실험적 기능 — 활성화 옵션을 켜야 사용 가능
